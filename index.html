Breast Histopathology Images Classification with CNNs and Variants¶

Kun Tao¶

Convolutional neural networks (CNNs) are well suited for image recognition. In this work, speeded up by GPU supported by the Google Colab, CNNs with/without Batch Normalization, Data Augmentation and Residual Connections built with Keras on TensorFlow are trained with 39393 subsampled, balanced non-IDC or IDC Breast Histopathology Images first, then applied to 13131 test images and obtained a test accuracy of 84.2%.

Using TensorFlow backend.

Train a convolutional neural network on the following dataset: https://www.kaggle.com/paultimothymooney/breast-histopathology-images

In order to load the dataset 'IDC_regular_ps50_idx5.zip' into Colab, I first uploaded it into my Google Drive, then downloaded and unzipped it to my newly created directory 'task3_data' in the 'Files' section of Colab.

Next, get filename lists 'images_class0' and 'images_class1'. As we see, there are 198738 images in class 0 and 78786 images in class 1. To avoid crashing the GPU after reading the images into Colab, I took a subsample from the original data and meanwhile kept the ratio of class 0 to class 1, that is, randomly read 66246 images from class 0 and 26262 images from class 1 to obtain (X_imgs, y) from which I further generated a balanced subsample (X_imgs_bal, y_bal) by using RandomUnderSampler. The balanced subsample includes 52524 images, and class 0 and class 1 each has 26262 images.

To make sure nothing was wrong during the above sampling preprocessing, two random images are respectively picked from the imbalanced X_imgs and the balanced X_imgs_bal, and displayed in Figure 2 and 3 below.

Then, I split (X_imgs_bal, y_bal) into X_train_imgs, X_test_imgs, y_train, y_test, and further applied necessary preprocessing to them. Again, to make sure nothing was worng, I randomly picked two images from X_train_imgs and displayed them in Figure 4 below.

To compare the effect of batch normalization, in 3.1 a), I first built a convolutional neural network (CNN) without batch normalization, then in 3.1 b), I developed a CNN with batch normalization. After trial and error, I set the 'filters' in the Conv2D as 8, and in the 'fit' method, I set batch_size=128, epochs=50 and validation_split=.2 for both models, and further changed the learning rate of the Adam optimizer from the default 0.001 to 0.0001. The combination of these settings not only greatly reduced the wild fluctuations of the validation curve in my initial experiments, but also speed up the training of the required CNN. The learning curves for these two models are shown in Figure 5 and Figure 6, and both look pretty reasonable.

The accuracy scores evaluated on ( X_test_imgs, y_test) are as follows:

CNN with/without Batch Normalization¶


CNN without Batch Normalization:

  • Test loss of CNN without Batch Normalization: 0.415
  • Test accuracy of CNN without Batch Normalization: 0.813

CNN with Batch Normalization:

  • Test loss of CNN with Batch Normalization: 0.389
  • Test accuracy of CNN with Batch Normalization: 0.838

As we see batch normalization does help here. After using it , the test accuracy is improved by 0.838 - 0.813 = 0.025.

CNN with Data Augmentation¶


In this subtask, settings in the ImageDataGenerator, fit_generator and evaluate_generator are chosen based on the examples shown at the Keras ' Image Preprocessing' webpage, as well as my own trial and error. Also, I used train_test_split to explicitly split the (X_train_imgs, y_train) into X_train_imgs_aug, X_val_imgs_aug, y_train_aug, y_val_aug, and the (X_val_imgs_aug, y_val_aug) was used for the val_generator which was further used in the fit_generator later. The test loss and accuracy of the data augmented by rotations, mirroring and other transformations are:

  • Test loss of CNN with Data Augmentation: 0.384
  • Test accuracy of CNN with Data Augmentation: 0.831

Compared with the CNN model without batch normalization, the test accuracy of the CNN with data augmentation is improved by 0.831 - 0.813 = 0.018. Compared to the CNN with batch normalization, the test accuracy of the CNN with augmented data is slightly lower.

Also, the learning curves of the CNN with Data Augmentation are shown in Figure 7.

Deep CNN with or without Residual Connections¶


First, I built a CNN model using Residual Connections which has 18 conv2d Layers and 9 max_pooling2d layers with skipping connections added, 1 flatten layer, 1dense layer, as well as the input layer and output layer. After trial and error, I set the 'filters' in the 'Conv2D' as 8, and also set batch_size=128, epochs=20 and validation_split=.2. These settings greatly speed up the training of my model and avoid the overfitting. The learning curves are shown in Figure 8 which looks well. The test loss and accuracy evaluated on the (X_test_imgs, y_test) are:

  • Test loss of a deep CNN with Residual Connections: 0.378
  • Test accuracy of a deep CNN with Residual Connections: 0.842

Then, I built another model with the same architecture but without using Residual Connections. The learning curves are shown in Figure 9 and the test loss and accuracy are:

  • Test loss of a deep CNN without Residual Connections: 0.693
  • Test accuracy of a deep CNN without Residual Connections: 0.500

As we see, the model without using Residual Connections only has a test accuracy of 0.5, which is equivalent to the performance of random guesses in terms of the balanced subsample data used for Task 3, in other words, it already lost the ability to learn.
